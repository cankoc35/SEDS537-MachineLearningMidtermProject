All experiments are implemented as Python scripts and are intended to be executed from the project root. The project environment uses Python 3.11 with dependencies installed from \texttt{requirements.txt}. The environment can be recreated with the following commands:

\begin{verbatim}
python3.11 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
\end{verbatim}

The shared random seed is defined as \texttt{RANDOM\_STATE = 42} in \texttt{src/common/config.py}. For Question 1, the same 80/20 train-test split is used across all regression experiments. The held-out test set is used only for final evaluation, while model selection for Ridge Regression is performed with 5-fold cross-validation on the training data.

Preprocessing steps are implemented inside scikit-learn pipelines where applicable. This ensures that transformations such as standard scaling and polynomial feature expansion are fit only on the training data and then applied to the test data, avoiding information leakage. Generated figures and metric tables are saved under \texttt{results/figures/} and \texttt{results/tables/} so that the report outputs can be reproduced by rerunning the corresponding scripts.

For Question 1, the full workflow can be reproduced with:

\begin{verbatim}
python -m src.q1_regression.run
\end{verbatim}

\subsubsection{Impact of Standard Scaling}

Standard scaling had little impact on predictive performance. Linear Regression produced almost identical RMSE, MAE, and $R^2$ values before and after scaling. This is expected because scaling changes the magnitude of the coefficients but does not substantially change the fitted linear relationship. Random Forest also showed almost no change, which is expected because tree-based models are generally not sensitive to feature scale.

For Ridge Regression, scaling changed the selected alpha value from 10.0 to 0.001, showing that feature scale affected the regularization process. However, the final test performance remained very close to the unscaled Ridge model. Therefore, standard scaling improved the comparability of feature scales and affected the Ridge hyperparameter selection, but it did not lead to a meaningful improvement in predictive performance for this dataset and split.

\subsubsection{Impact of Polynomial Features}

Degree-2 polynomial features improved the Linear Regression model compared with the original baseline. The polynomial linear model reached an RMSE of 0.6814, MAE of 0.4670, and $R^2$ of 0.6457, while the original Linear Regression model had an RMSE of 0.7456, MAE of 0.5332, and $R^2$ of 0.5758. This suggests that the California Housing dataset contains nonlinear relationships and feature interactions that a simple linear model cannot capture.

Polynomial Ridge Regression did not improve performance. Its $R^2$ was 0.5733 and the selected alpha was 1000.0, indicating that strong regularization was selected after the polynomial expansion. This likely reduced the benefit of the additional polynomial terms.

The Polynomial Random Forest Regressor achieved an $R^2$ of 0.7991, which was slightly worse than the original Random Forest result of 0.8062. This is reasonable because Random Forest can already model nonlinear relationships and interactions through tree splits, so manually adding polynomial features does not necessarily help and can add unnecessary feature complexity. Overall, polynomial features were useful for the linear model, but Random Forest remained the strongest model.

Random Forest achieved the best predictive performance, with the lowest RMSE and MAE and the highest $R^2$ score of 0.8062. This suggests that nonlinear relationships and feature interactions are important for the California Housing dataset. The trade-off is that Random Forest is less directly interpretable than the linear models because predictions are based on many decision trees rather than a small set of coefficients. Overall, Random Forest is the preferred model for prediction, while Linear Regression and Ridge Regression are more useful for simpler interpretation.

\subsection{Question 2: Classification - Handling Class Imbalance}

\subsubsection{Dataset Description}

For Question 2, the Credit Card Fraud Detection dataset was used. The dataset contains 31 columns in total: 28 anonymized PCA-transformed numerical features (\texttt{V1} to \texttt{V28}), the \texttt{Time} feature, the \texttt{Amount} feature, and the target variable \texttt{Class}. The target is binary, where \texttt{Class = 0} represents a legitimate transaction and \texttt{Class = 1} represents a fraudulent transaction.

The anonymized variables \texttt{V1} to \texttt{V28} do not have direct real-world meanings because they were transformed using Principal Component Analysis (PCA) to protect sensitive financial information. However, they still retain useful information for fraud detection. The \texttt{Amount} column represents the transaction amount, while \texttt{Time} records the number of seconds elapsed since the first transaction in the dataset.

\subsubsection{Class Distribution}

The dataset is highly imbalanced. There are 284,315 legitimate transactions (\(99.8273\%\)) and only 492 fraudulent transactions (\(0.1727\%\)). This confirms that the fraud class is extremely underrepresented compared with the legitimate class.

Such an imbalance makes accuracy alone a misleading evaluation metric, because a classifier could achieve very high accuracy by predicting almost every transaction as legitimate. For this reason, Question 2 focuses on precision, recall, F1-score, ROC-AUC, confusion matrix analysis, and resampling methods such as SMOTE and random undersampling.

\subsubsection{Baseline Models}

An 80/20 stratified train-test split was used so that the severe class imbalance was preserved in both partitions. Three baseline classifiers were then trained on the original imbalanced training data: Logistic Regression, Random Forest, and XGBoost. Their performance on the held-out test set is summarized in Table~\ref{tab:q2-baseline-models}.

\begin{table}[ht]
    \centering
    \caption{Question 2 baseline classifier comparison on the imbalanced test set.}
    \label{tab:q2-baseline-models}
    \begin{tabular}{lrrrr}
        \toprule
        Model & Precision & Recall & F1-score & ROC-AUC \\
        \midrule
        Logistic Regression & 0.8267 & 0.6327 & 0.7168 & 0.9605 \\
        Random Forest & 0.9419 & 0.8265 & 0.8804 & 0.9623 \\
        XGBoost & 0.9277 & 0.7857 & 0.8508 & 0.9616 \\
        \bottomrule
    \end{tabular}
\end{table}

The baseline results show that the tree-based models perform substantially better than Logistic Regression on this dataset. Logistic Regression achieves a strong ROC-AUC, but its recall is only 0.6327, meaning that it misses a relatively large portion of fraudulent transactions. This is a common issue in highly imbalanced problems, where a simple linear model can separate many normal transactions well but still fail to recover enough minority-class examples.

Random Forest produced the strongest baseline result overall, with the highest precision, recall, F1-score, and ROC-AUC among the three models. Its confusion matrix shows only 17 false negatives and 5 false positives on the test set, which indicates a better balance between fraud detection and avoiding unnecessary fraud alerts. XGBoost also performed strongly and clearly outperformed Logistic Regression, but it remained slightly behind Random Forest in recall and F1-score for this baseline configuration.

At this stage, the main finding is that model choice already matters even before applying resampling. However, because the dataset remains extremely imbalanced, the next step is to test whether SMOTE or random undersampling can further improve minority-class detection, especially recall, without creating too many false positives.

\subsubsection{Resampling Results}

SMOTE and random undersampling were both applied only to the training set after the stratified split, while the test set remained unchanged. SMOTE produced a balanced training set with 227,451 legitimate transactions and 227,451 fraudulent transactions. Random undersampling reduced the majority class down to the minority class size, producing a much smaller balanced training set with 394 legitimate transactions and 394 fraudulent transactions. Table~\ref{tab:q2-resampling-comparison} compares the baseline, SMOTE, and random undersampling results on the held-out test set.

The resampling experiments show that both methods strongly shift the precision-recall trade-off toward higher recall. For Logistic Regression, SMOTE increased recall from 0.6327 to 0.8980, and random undersampling increased it further to 0.9184. However, precision dropped sharply to 0.1341 with SMOTE and to 0.0516 with undersampling. This caused the F1-score to fall well below the baseline value. In other words, the model found more fraud cases, but it also produced many more false alarms.

For Random Forest, SMOTE did not improve the overall balance of performance. Recall remained at 0.8265, while precision dropped from 0.9419 to 0.8265, lowering the F1-score from 0.8804 to 0.8265. Random undersampling pushed recall to 0.9184, but precision collapsed to 0.0423 because the model flagged far too many legitimate transactions as fraud. As a result, the undersampled Random Forest performed much worse than both the baseline and SMOTE versions under the F1-score criterion.

For XGBoost, SMOTE increased recall from 0.7857 to 0.8673 and improved ROC-AUC from 0.9616 to 0.9820. However, precision decreased from 0.9277 to 0.7456, and the F1-score fell from 0.8508 to 0.8019. Random undersampling again forced recall up to 0.9184, but precision dropped to 0.0348 and the F1-score fell to 0.0671. This indicates that the undersampled XGBoost model became far too aggressive in predicting fraud.

Overall, the current results indicate that SMOTE is useful when the goal is to recover more fraud cases, especially for Logistic Regression and XGBoost, but it does not automatically improve the most balanced model. Random undersampling increased recall for all three classifiers, yet it damaged precision so severely that all three undersampled models had very poor F1-scores. Among all experiments completed so far, the baseline Random Forest still provides the strongest overall trade-off between fraud detection and false alarms.

\subsubsection{ROC Curve and Error Analysis}

Figure~\ref{fig:q2-random-forest-roc} shows the ROC curve for the baseline Random Forest model, which produced the strongest overall classification result among the baseline, SMOTE, and random undersampling experiments.

\end{figure}

The ROC curve plots the true positive rate against the false positive rate across different decision thresholds. A curve closer to the top-left corner indicates better class separation, while the diagonal reference line represents random guessing. The baseline Random Forest achieved an ROC-AUC of 0.9623, which indicates strong discrimination between legitimate and fraudulent transactions.

This result is consistent with the confusion matrix and F1-score analysis. The baseline Random Forest combines high precision (0.9419) with strong recall (0.8265), so it catches most fraud cases while keeping false alarms low. Compared with the resampled models, its ROC-AUC is not always the highest, but its threshold-based performance at the default decision rule is more balanced. This is why it remains the strongest overall model for the current fraud detection setup.

\subsubsection{Discussion}

In the credit card fraud detection setting, there is a clear trade-off between precision and recall. A model with higher recall can identify more fraudulent transactions, but this often comes at the cost of lower precision, which means more legitimate transactions are incorrectly flagged as fraud. In practice, this trade-off matters because the two types of classification error have different business consequences.

For this dataset, a false negative is generally the more costly error. A false negative means that a fraudulent transaction is missed and allowed to pass as legitimate, which can lead to direct financial loss, customer harm, and delayed fraud response. A false positive is also undesirable because it may block a legitimate transaction, inconvenience the customer, and create unnecessary review work, but this cost is usually smaller than failing to stop an actual fraudulent charge.

The resampling experiments illustrate this trade-off clearly. SMOTE and random undersampling both pushed recall upward, but they did so by sharply increasing false positives and reducing precision. This behavior may be acceptable in some fraud monitoring systems where catching as many fraud cases as possible is the top priority. However, in the current experiment, the loss of precision was too severe, especially for the undersampled models, which led to much weaker F1-scores.

Overall, the baseline Random Forest is the most appropriate model among the tested configurations. It achieves the best balance between fraud detection and false alarms, with high precision, strong recall, the highest F1-score, and a strong ROC-AUC. Therefore, it is the preferred model for this dataset under a balanced evaluation criterion, while a higher-recall resampled model could still be considered in a setting where minimizing false negatives is the primary objective.

The full dataset contains 70,000 samples. The feature matrix therefore has shape \(70{,}000 \times 784\), and the label vector has shape \(70{,}000\). The labels cover all ten digit classes from 0 to 9, and pixel intensities range from 0 to 255.

\subsubsection{Discussion}

PCA and t-SNE serve different purposes, and the Question 3 results make that distinction clear. PCA is a linear dimensionality reduction method that is fast, reproducible, and easy to extend to new data. It is useful both for compact feature reduction and for giving a coarse visual summary of the global structure of the dataset. Its weakness is that the 2D embedding does not separate digit classes cleanly, because the first two principal components cannot capture all of the nonlinear variation in handwritten digits.

t-SNE is much stronger for visualization. The 2D t-SNE plots are expected to show tighter and more clearly separated local digit clusters than PCA because the method is designed to preserve neighbourhood relationships. This makes t-SNE especially useful when the goal is to inspect whether similar classes form local groups in the latent space. Its weaknesses are that it is computationally expensive, sensitive to hyperparameters such as perplexity, and not naturally designed as a reusable feature extractor for new unseen samples.

For downstream classification, PCA is the more practical reduction method because it produces a stable lower-dimensional representation that can easily be applied to additional data. In this experiment, PCA 50D achieved nearly the same k-NN accuracy as the best t-SNE result while remaining more efficient and easier to interpret as a dimensionality reduction pipeline. Overall, t-SNE is preferable for visualization, while PCA is the more suitable general-purpose method for dimensionality reduction and classification workflows.

\subsection{Question 4: Clustering - Unsupervised Learning}

\subsubsection{Dataset Description}

For Question 4, the Mall Customer Segmentation dataset is used. The dataset contains 200 customer records and 5 columns: \texttt{CustomerID}, \texttt{Gender}, \texttt{Age}, \texttt{Annual Income (k\$)}, and \texttt{Spending Score (1-100)}. The goal is to discover natural customer groups without using any target labels, which makes this an unsupervised learning task.

The \texttt{CustomerID} column is only an identifier and does not carry behavioural information, so it is not useful as a clustering feature. The remaining variables describe demographic and spending characteristics that can support customer segmentation. In particular, annual income and spending score are expected to be important for identifying customer groups with different purchasing behaviour.

The dataset does not contain missing values, which simplifies preprocessing. However, the \texttt{Gender} feature is categorical and must be encoded before clustering, and the numeric variables should be scaled so that no single feature dominates the distance-based clustering algorithms.

\subsubsection{Preprocessing}

Before clustering, the \texttt{CustomerID} column was removed because it is only an identifier and does not describe customer behaviour. The \texttt{Gender} feature was encoded numerically so that it could be used by the distance-based clustering algorithms. Finally, all clustering features were standard-scaled so that variables measured on different ranges, such as age, income, and spending score, would contribute more fairly to the distance calculations. The final processed feature matrix has shape \(200 \times 4\).

\subsubsection{K-Means Clustering}

K-Means was first evaluated for \(k=2\) through \(k=10\). For each value of \(k\), inertia was recorded for the elbow method and silhouette score was computed to assess cluster separation. Figure~\ref{fig:q4-kmeans-selection} summarizes these results.

\begin{figure}[ht]
    \centering
    \includegraphics[width=\textwidth]{../../results/figures/q4/kmeans_model_selection.png}
    \caption{K-Means model selection using the elbow method and silhouette score.}
    \label{fig:q4-kmeans-selection}
\end{figure}

The inertia curve decreases steadily as \(k\) increases, which is expected because additional clusters reduce within-cluster variation. However, the curve starts to flatten around \(k=5\) to \(k=6\), which suggests a practical elbow in that region. The silhouette score rises from about 0.252 at \(k=2\) to about 0.421 at \(k=10\), but this does not automatically mean that \(k=10\) is the best practical choice because larger \(k\) values can split the customers into overly fine groups that are harder to interpret.

Based on the current plot, \(k=5\) is selected as the working K-Means choice for the next step of the analysis. It is close to the elbow region and gives a more balanced customer segmentation than simply choosing the largest tested \(k\). If the goal were only to maximize silhouette score within the tested range, \(k=10\) would be preferred, but for this report \(k=5\) is the more defensible compromise between compactness and interpretability.

\subsubsection{Agglomerative Clustering}

Agglomerative Clustering was applied with Ward linkage and \(n\_clusters=5\) so that it could be compared directly with the selected K-Means solution. Figure~\ref{fig:q4-agglomerative-pca} shows the resulting cluster assignments in a 2D PCA projection.

\subsubsection{Cluster Interpretation}

The cluster summary indicates five interpretable customer groups. Cluster 0 contains relatively high-income and high-spending customers, which makes it the most valuable segment from a business perspective. Cluster 1 also has high income but very low spending, suggesting a cautious or under-engaged customer group. Cluster 3 is the youngest segment, with lower income but still relatively high spending, which may reflect younger active shoppers.

Clusters 2 and 4 are older, moderate-income, lower-spending groups. In the current K-Means result, Cluster 2 is entirely female and Cluster 4 is entirely male. This suggests that the encoded \texttt{Gender} variable has a noticeable influence on the clustering outcome. Therefore, the discovered groups should be interpreted as a mixture of demographic and spending behaviour rather than as pure spending-based segments only.

\subsubsection{Discussion}

The Question 4 results show that K-Means is the strongest method for this particular dataset. The customer groups are reasonably compact, interpretable, and aligned with typical mall-segmentation patterns such as high-income/high-spending and high-income/low-spending customers. Agglomerative Clustering gives a similar but slightly weaker segmentation, while DBSCAN performs poorly under the tested setting because it mainly splits customers into two broad density groups and is strongly affected by the encoded gender feature.

DBSCAN is also clearly the most sensitive method to its parameters. Small changes to \(\varepsilon\) or \texttt{min\_samples} can change the number of clusters and the number of noise points substantially. This sensitivity is one of its main weaknesses on small tabular datasets like Mall Customers, especially when the underlying groups are not very irregular in shape. By contrast, K-Means and Agglomerative Clustering are more stable here because the data structure is relatively compact and cluster-like after scaling.

In terms of cluster shape, K-Means and Ward-linkage Agglomerative both prefer compact groups, which matches this dataset reasonably well. DBSCAN is more appropriate when clusters have irregular shapes or when explicit outlier detection is important. For this report, however, K-Means provides the most useful final clustering solution.
